⏺ Technical challenges for this build:

  DEX parsing
  - Format = segment-based, *-delimited, / line terminator. Not CSV, not JSON.
  - Variable-length PA segments (1..N lanes per file). Need loop, not fixed
  schema.
  - Field indexing per NAMA spec: ID1 field 01 = serial, VA1 field 01 = paid
  vends, PA1 01/02 = product/price, PA2 01/02 = vends/sales.
  - Money fields = integer cents, not decimal. Convert on store (÷100) or keep
  as bigint.
  - DEX timestamp not in one segment — usually G85/DXE or derived from ID1 date
  + time. Need pick right source for DEXDateTime.

  MAUI client
  - Basic auth header: Authorization: Basic base64(user:pass). Body = raw DEX
  text, Content-Type: text/plain.
  - Emulator→host networking: Android emulator 10.0.2.2, iOS sim localhost.
  Clear-text HTTP needs usesCleartextTraffic + iOS ATS exception if not HTTPS.
  - Hardcoded DEX strings: escape \r\n correctly, preserve segment terminators.

  API auth
  - Basic auth middleware or AuthenticationHandler<BasicOptions>. Constant-time
  compare to avoid timing leak. Creds from appsettings.json + IOptions.
  - Return 401 + WWW-Authenticate: Basic realm="vdi-dex" on fail.

  DB schema + FK
  - DEXMeter PK = identity or (Machine, DEXDateTime) composite. Spec says
  DEXDateTime unique per machine → composite natural key works but identity int
  PK simpler for FK.
  - DEXLaneMeter.DEXMeterId FK → DEXMeter.Id. Index on FK.
  - Types: MachineSerialNumber nvarchar(20), money as decimal(12,2) or bigint
  cents, NumberOfVends int, Price decimal(10,2).

  Stored proc path
  - Option A (T-SQL parse): painful. STRING_SPLIT no ordinal pre-SQL2022, nested
   splits ugly. Skip.
  - Option B (C# parse, 2 procs): cleaner. But lane insert = N round-trips
  unless use TVP. TVP (dbo.LaneMeterType) = one call, atomic, fast.
  - Wrap both in single transaction so partial inserts impossible.

  Concurrency + speed
  - Async all the way (ReadAsStringAsync, SqlConnection.OpenAsync,
  ExecuteNonQueryAsync).
  - Connection pooling default on — don't disable.
  - Duplicate submission: same (Machine, DEXDateTime) twice → UNIQUE constraint
  + catch 2627/2601, return 409.
  - Large body guard: MaxRequestBodySize cap.

  Edge cases
  - Missing VA101 (machine never sold) → null vs 0.
  - PA rows without matching PA2 (configured but no sales) → left-join pair by
  lane index.
  - CRLF vs LF line endings between the two sample files.
